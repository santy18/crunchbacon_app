# Contributing to CrunchBacon

We love your input! We want to make contributing to CrunchBacon as easy and transparent as possible, whether it's:

- Reporting a bug
- Discussing the current state of the code
- Submitting a fix
- Proposing new features
- Becoming a maintainer

## Development Process

1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. If you've changed APIs, update the documentation.
4. Ensure the test suite passes.
5. Make sure your code lints.
6. Issue that pull request!

## Coding Style

- Use Python 3.11+ (backend)
- Follow PEP 8 (backend)
- Use React with functional components (frontend)
- Standard Prettier/ESLint rules for frontend
